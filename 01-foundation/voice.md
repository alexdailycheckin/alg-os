# Voice

This is a template. The intake produces a filled version at `/workspace/foundation/voice.md`. All skills read that file before generating output.

## Template

```
# Voice - <Operator Name>

## English variant

<British | American>

## Default rules

- No em dashes. No other dashes besides "-".
- Three sentences per paragraph maximum.
- Digits not words for numbers.
- Operator tone, not marketing tone.
- State things directly.
- No "that's not X, that's Y" reframing.

## Operator overrides

<list any rules the operator overrides, with rationale. Empty if none.>

## Banned words

<list of words and phrases this brand does not use. Reasons in parentheses where useful.>

Examples:
- "leverage" (overused)
- "best-in-class" (adjective without proof)
- "synergy" (corporate jargon)

## Tone notes

<3 to 5 sentences describing how the brand sounds. Avoid adjectives that mean nothing ("approachable yet professional"). Prefer comparisons ("sounds like a senior operator briefing a board, not a marketer pitching a product").>

## Output formats

<Default formats this brand uses. Useful for skill output formatting.>

Examples:
- Email signatures: <format>
- Headline style: <Sentence case | Title Case>
- Punctuation defaults: <UK conventions | US conventions>
```

## Quality bar

The filled voice.md is defensible when:

- English variant is unambiguous.
- Banned-word list is honest. "We ban nothing" is a refusal to take a stance, which is itself a stance worth challenging.
- Tone notes describe a real voice, not an aspirational adjective list.
- A skill running against this file produces output the operator immediately recognises as "us."
