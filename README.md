# ALG OS

**Agent-Led Growth Operating System.** A brand-neutral, tool-agnostic framework any operator can fork and run on their own business.

## What this is

ALG OS is a framework for building outreach machines, not content production systems. The output is a GTM motion where unit economics improve quarter on quarter because the artefacts compound and the data compounds.

It is independent of any specific company. The recommendation engine is open and shows multiple options per agent category. ALG OS is not a kingmaker.

## Why it exists

Cold outbound is converging. Apollo, Clay, GPT, the same playbook. Reply rates have collapsed because buyers' defences have evolved faster than the offence.

Doing the same thing better is a losing trade. ALG OS defines a different category of motion: agent-led, where agents do not just power the growth motion, they ARE the growth motion.

## Core idea

The product produces the marketing. The marketing demonstrates the product. They are the same thing.

Every ALG-designed motion follows the same shape:

1. The signature artefact reveals an external truth (a leak, a gap, a benchmark).
2. The buyer asks the natural next question.
3. The operator's product or a recommended tool answers it with a low-commitment first step.
4. Full deployment closes the gaps the first step surfaced.

## Two modes

ALG OS operates in one of two modes per operator, selected by the intake diagnostic.

**Product-recursive.** For product companies. Agents executing the GTM motion are powered by the company's own product. Every demo of the motion is a demo of the product.

**Tool-agnostic.** For service companies, agencies, consultancies, and any operator without a recursive product. ALG OS recommends best-in-class tools per agent across three tiers.

## How to run it

Once Build 3 is complete, the operator path is:

1. Fork this repo and clone locally.
2. Open Claude Desktop, create a Project pointing at the cloned folder. Or use Claude Code from the terminal.
3. Say "run intake." The sparring layer interrogates your business until inputs clear the quality bar, then generates Layer 1 foundation files into `/workspace/`.
4. Run the five canonical skills against your foundation. Output saves to `/workspace/execution/`.
5. When framework upgrades are available, run `./alg-upgrade`. Workspace stays untouched.

Detailed operator instructions ship with Build 3. See [`foundation.md`](./foundation.md) section 12 for the build sequence.
